# [PRD V2] 향수 이커머스 플랫폼 제품 요구사항 문서

## 1. 개요 및 브랜드 가이드

### 1) 한 줄 소개
* 관리자, 입점사, 소비자를 연결하여 다양한 향수를 편리하게 거래하고 관리할 수 있는 역할 기반의 향수 전문 이커머스 플랫폼

### 2) 서비스 이름 및 브랜드 스타일 가이드
* **업체명 (서비스 이름)**: **센트아틀리에 (ScentAtelier)**
  * *의미*: 조향사의 작업실(Atelier)처럼 깊이 있고 전문적인 향수 경험을 제공한다는 의미
* **컨셉 컬러 (Color Palette)**:
  * **Primary (메인)**: 깊고 고급스러운 딥 우디 네이비 (`#1E293B`)
  * **Secondary (포인트)**: 따뜻하고 세련된 미스틱 앰버 골드 (`#D97706`)
  * **Background**: 정갈하고 깨끗한 리넨 화이트 (`#F8FAFC`)
* **디자인 스타일**: 
  * 미니멀리즘 and 하이엔드 갤러리 감성을 결합한 레이아웃
  * 넓은 여백과 고해상도 보틀(Bottle) 이미지를 강조하여 시각적인 향의 깊이를 전달하는 스타일

---

## 2. 비즈니스 목표 및 타겟 사용자

### 1) 목표 (3개)
1. **역할별 최적화된 인터페이스 제공**: 소비자, 입점사, 관리자가 각각의 목적에 맞게 효율적으로 시스템을 사용할 수 있도록 분리된 환경을 구축합니다.
2. **안정적인 결제 및 주문 프로세스 확보**: 토스 페이 연동을 통해 소비자가 끊김 없이 안전하게 주문 및 결제를 완료할 수 있도록 합니다.
3. **신뢰 중심의 커뮤니티 및 관리 환경 조성**: 리뷰, 신고, 1:1 문의 시스템을 통해 투명한 상품 정보와 신속한 고객 지원을 제공합니다.

### 2) 타겟 사용자
* **소비자**: 자신에게 맞는 향수를 검색, 비교하고 편리하게 구매 및 배송 추적을 원하는 개인 사용자
* **입점사**: 자체 브랜드 또는 수입 향수를 플랫폼에 등록하여 판매하고, 주문 및 정산(전자세금계산서 등)을 관리하고자 하는 판매자
* **관리자**: 전체 회원, 입점사, 상품, 주문 현황을 모니터링하고 공지사항 및 신고 내역을 관리하는 플랫폼 운영자

## 3. 핵심 기능 (사용자 행동 단위)

1. **역할 기반 인터페이스 분리 (필수)**
   * *소비자*: 메인 페이지에서 향수를 탐색하고 마이페이지에서 주문 내역을 조회한다.
   * *입점사*: 별도 파트너 센터에 로그인하여 상품을 입점 신청하고 전용 대시보드에서 매출 현황을 확인한다.
   * *관리자*: 백오피스 시스템에 접근하여 전체 회원 상태를 제어하고 신고된 콘텐츠를 블라인드 처리한다.
2. **간편 가입 및 로그인 (필수)**
   * *소비자*: 카카오/네이버 버튼을 클릭하여 별도의 복잡한 절차 없이 3초 만에 회원가입 및 로그인을 완료한다.
3. **토스 페이 통합 주문/결제 (필수)**
   * *소비자*: 장바구니에서 구매할 향수를 선택한 뒤, 토스 페이 인증 창을 통해 생체인식 또는 비밀번호 입력만으로 결제를 완료한다.
4. **실시간 배송 상태 추적 (선택)**
   * *소비자*: 마이페이지 주문 상세 내역에서 택배사 API가 제공하는 실시간 이동 경로와 현재 배송원 위치를 확인한다.
5. **CS 및 운영 관리 (선택)**
   * *소비자*: 구매한 향수에 대한 리뷰를 작성하거나 불만 사항을 1:1 문의로 접수하고, 유해한 리뷰 발견 시 신고 버튼을 누른다.

## 4. 화면 목록 및 필수 UI 요소 (3개씩)

1. **메인 / 상품 목록 페이지 (소비자용)**
   * *필수 UI 요소 1*: **향 계열 필터 칩 (Filter Chips)** - 우디, 플로럴, 시트러스 등 향 유형별로 상품을 빠르게 분류하는 상단 필터
   * *필수 UI 요소 2*: **고해상도 썸네일 그리드 (Product Grid)** - 앰버 골드와 네이비 톤이 어우러진 감성적인 향수 보틀 이미지와 가격 정보 카드
   * *필수 UI 요소 3*: **실시간 베스트 랭킹 슬라이더 (Carousel)** - 소비자들이 가장 많이 찾는 인기 향수를 순위별로 노출하는 배너 영역
2. **상품 상세 페이지 (소비자용)**
   * *필수 UI 요소 1*: **향 노트 그래프 (Scent Note Chart)** - 탑, 미들, 베이스 노트를 시각적으로 보여주는 UI 컴포넌트
   * *필수 UI 요소 2*: **구매/장바구니 고정 플로팅 바 (Sticky Bottom Bar)** - 스크롤 위치와 상관없이 언제든 바로 구매하거나 장바구니에 담을 수 있는 하단 고정 바
   * *필수 UI 요소 3*: **포토 리뷰 및 별점 요약 공간 (Review Summary)** - 소비자들이 등록한 실제 향수 사진과 별점 평균을 한눈에 보는 레이아웃
3. **장바구니 / 결제 페이지 (소비자용)**
   * *필수 UI 요소 1*: **입점사별 상품 묶음 카드 (Vendor Grouping)** - 배송비가 따로 계산되는 입점사별로 상품을 그룹화하여 보여주는 리스트
   * *필수 UI 요소 2*: **배송지 입력 및 토스페이 전용 버튼 (Payment CTA)** - 원클릭으로 토스 페이 결제 모달을 띄우는 브랜드 컬러 기반의 대형 버튼
   * *필수 UI 요소 3*: **최종 결제 금액 요약 박스 (Summary Box)** - 총 상품 금액, 배송비, 할인 금액, 최종 납부 금액을 직관적으로 보여주는 우측 고정 박스
4. **입점사 어드민 페이지 (입점사용)**
   * *필수 UI 요소 1*: **주문/배송 상태 변경 매트릭스 (Status Dashboard)** - 신규주문, 배송준비, 배송중, 완료 건수를 숫자로 보여주고 원클릭으로 상태를 업데이트하는 대시보드
   * *필수 UI 요소 2*: **향수 사양 등록 폼 (Product Form)** - 용량(ml), 향 계열, 지속력 등 향수 특화 정보를 입력하는 입력 필드 세트
   * *필수 UI 요소 3*: **전자세금계산서 발행 요청 버튼 (Tax Invoice Button)** - 정산 주기별로 세금계산서 발행을 플랫폼 관리자에게 요청하고 상태를 확인하는 테이블
5. **통합 관리자 슈퍼 어드민 페이지 (관리자용)**
   * *필수 UI 요소 1*: **실시간 신고 모니터링 큐 (Report Queue)** - 소비자나 입점사가 접수한 신고 글(리뷰/문의 등)을 빠르게 확인하고 블라인드 처리하는 관리 목록
   * *필수 UI 요소 2*: **FAQ 및 공지사항 에디터 (Rich Text Editor)** - 플랫폼 공지 및 자주 묻는 질문을 텍스트와 이미지로 편집하여 게시하는 에디터 화면
   * *필수 UI 요소 3*: **입점사 승인/반려 토글 (Vendor Management)** - 신규 신청한 입점사의 서류를 확인하고 승인 또는 반려 사유를 입력하는 관리 시스템

## 5. 사용자 흐름 (소비자의 상품 구매 흐름)

1. 소비자가 **소셜 로그인**을 통해 사이트에 접속합니다.
2. 상품 목록 또는 검색을 통해 원하는 **향수 상세 페이지**로 이동합니다.
3. 상품 옵션을 선택한 후 **장바구니**에 담습니다.
4. 장바구니에서 주문하기를 눌러 배송지를 입력하고 **토스 페이**로 결제를 진행합니다.
5. 결제 완료 후 **주문 완료 페이지**를 확인하고, 이후 마이페이지에서 **배송 추적**을 시작합니다.

## 6. 데이터 모델 초안 (필드만)

* **User (사용자)**: 사용자ID, 이메일, 비밀번호, 이름, 휴대폰번호, 역할구분(소비자/입점사/관리자), 가입일시
* **Product (상품)**: 상품ID, 입점사ID, 상품명, 브랜드명, 가격, 재고수량, 향계열(우디/플로럴 등), 상품설명, 대표이미지URL, 등록일시
* **Order (주문)**: 주문ID, 소비자ID, 총결제금액, 배송지주소, 수령인이름, 수령인연락처, 주문상태(결제완료/배송중/배송완료), 결제승인번호(토스), 주문일시
* **Review (리뷰)**: 리뷰ID, 상품ID, 소비자ID, 별점, 리뷰내용, 첨부이미지URL, 신고여부(True/False), 등록일시

## 7. 추가로 확인할 질문 10개

1. **소셜 로그인 대상**: 카카오, 네이버, 구글, 애플 중 어떤 소셜 로그인 매체를 필수적으로 먼저 연동해야 하나요?
2. **입점사 입점 프로세스**: 입점사 회원가입 시 관리자의 수동 승인 절차가 필요한가요, 아니면 가입 즉시 상품 판매가 가능한가요?
3. **정산 및 세금계산서**: 전자세금계산서 API 연동은 입점사가 소비자에게 발행하는 용도인가요, 아니면 플랫폼이 입점사에게 수수료를 청구하는 용도인가요?
4. **배송 추적 API**: 연동하고자 하시는 특정 택배사가 지정되어 있나요, 아니면 통합 배송 추적 서비스(예: 커스텀 API 등)를 사용하실 예정인가요?
5. **향수만의 특화 필드**: 향수 검색/필터링을 위해 '노트(탑/미들/베이스)', '향 계열(우디, 플로럴 등)', '성별(남성/여성/공용)' 같은 특화 데이터 필드가 필요한가요?
6. **리뷰 및 신고 정책**: 신고가 접수된 리뷰나 상품은 관리자 승인 전까지 즉시 블라인드 처리(숨김)되어야 하나요?
7. **1:1 문의 범위**: 소비자의 1:1 문의는 '플랫폼 관리자'에게 가나요, 아니면 해당 상품을 판매한 '입점사'에게 직접 전달되나요?
8. **장바구니 정책**: 서로 다른 입점사의 상품을 한 장바구니에 담아 한 번에 통합 결제할 수 있도록 지원해야 하나요?
9. **할인 및 프로모션**: 첫 버전에서 쿠폰, 적립금, 혹은 상품 할인율 적용 기능이 포함되어야 하나요?
10. **재고 관리 알림**: 입점사 상품의 재고가 소진되었을 때 품절 처리 방식이나 입점사 대상 알림(SMS/메일) 기능이 필요한가요?
